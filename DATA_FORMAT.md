# GIT Dataset Format Specification (Reverse Engineering Report)

This document details the exact layout, fields, shapes, and consumption patterns of the PyTorch Geometric (`torch_geometric.data.Data`) datasets expected by the GIT codebase.

---

## 1. Trace of the Dataset Loading Pipeline

When running pretraining or fine-tuning, the data loading flows through the following path:
1. **Execution Entry**: `pretrain.py` or `finetune.py` starts and calls `unified_data(params)` (for pretraining) or `get_data(params)` (for fine-tuning).
2. **Data Fetching**: `data/finetune_data.py` (via the `single_graph` function) loads the processed PyG dataset from the path:
   `cache_data/<dataset_name>/processed/geometric_data_processed.pt` using `torch.load(path)[0]`.
3. **Graph Preparation**: The dataset is converted to undirected using `ToUndirected()(data)`.
4. **Virtual Node Augmentation**: In `data/pretrain_data.py`, `VirtualNodeAugmentor` dynamically appends task-tree virtual nodes to `data.x` and connects them using custom edges, updating `data.edge_index` and padding `data.node_text_feat` with a zero row representing the virtual nodes.
5. **Batching**: The individual graphs are grouped into a disjoint batch using `Batch.from_data_list`.
6. **Loader Generation**: `NeighborLoader` is instantiated to sample neighborhood mini-batches during training.

---

## 2. Field-by-Field Analysis

Below is the exhaustive specification of every field utilized during the loading and training pipelines:

### 1. `node_text_feat`
* **Where it is created**: Generated by encoding normalized node descriptions using a SentenceTransformer (e.g., `BAAI/bge-base-en-v1.5`).
* **Where it is loaded**: Loaded in `data/finetune_data.py` (`single_graph`) and `data/pretrain_data.py` (`unified_data`).
* **Where it is consumed**: Consumed in GNN encoders (`model/encoder.py`) as the initial feature matrix of nodes.
* **Mandatory**: Yes.
* **Tensor Dtype**: `torch.float32`
* **Tensor Shape**: `[num_nodes, embedding_dim]` (e.g., `[num_nodes, 768]`).
* **Semantic Meaning**: Dense semantic representation of each node's text summary.
* **Equivalent Available**: Yes, we have `node_embeddings.npy` in `custom_pretrain/embeddings/`.

### 2. `class_node_text_feat`
* **Where it is created**: Generated by encoding the natural-language description of target categories (classes).
* **Where it is loaded**: Loaded in `data/finetune_data.py` (`single_graph`).
* **Where it is consumed**: Consumed in supervised fine-tuning/pretraining loss calculations (`task/node.py` and `task/edge.py`), where the integer labels in `data.y` are mapped to their corresponding class description embeddings to compute MSE loss.
* **Mandatory**: Yes, mandatory for SFT and zero-shot evaluation.
* **Tensor Dtype**: `torch.float32`
* **Tensor Shape**: `[num_classes, embedding_dim]` (e.g., `[2, 768]` or `[4, 768]`).
* **Semantic Meaning**: Semantic representations of the class targets, aligning them in the shared latent space.
* **Equivalent Available**: Yes, we have raw text descriptions in `class_text.json` which can be encoded on the fly.

### 3. `edge_text_feat`
* **Where it is created**: Generated by encoding edge descriptions.
* **Where it is loaded**: Loaded in `data/finetune_data.py` (`temporal_graph` loading).
* **Where it is consumed**: Consumed in edge classification tasks (`task/edge.py`) as `edge_attr`.
* **Mandatory**: Only mandatory for edge classification/temporal tasks.
* **Tensor Dtype**: `torch.float32`
* **Tensor Shape**: `[num_edges, embedding_dim]` (e.g., `[num_edges, 768]`).
* **Semantic Meaning**: Dense semantic representation of the transaction or interaction details.
* **Equivalent Available**: Yes, we have raw text in `edge_text.json`.

### 4. `edge_attr`
* **Where it is created**: Duplicated from `edge_text_feat` in `task/edge.py` or stored directly in the `Data` object.
* **Where it is loaded**: Loaded in `task/edge.py`.
* **Where it is consumed**: Consumed in `MySAGEConv.message` inside `model/encoder.py` where neighbors features are updated: `(x_j + edge_attr).relu()`.
* **Mandatory**: Yes, if message passing uses edge attributes.
* **Tensor Dtype**: `torch.float32`
* **Tensor Shape**: `[num_edges, embedding_dim]` (e.g., `[num_edges, 768]`).
* **Semantic Meaning**: Edge feature matrix used in message passing convolutions.
* **Equivalent Available**: Yes, same as `edge_text_feat`.

### 5. `edge_index`
* **Where it is created**: Built from the graph connections.
* **Where it is loaded**: Loaded in `pretrain.py`, `data/pretrain_data.py`, and `task/node.py`.
* **Where it is consumed**: Consumed in GNN convolutions to pass messages.
* **Mandatory**: Yes.
* **Tensor Dtype**: `torch.long`
* **Tensor Shape**: `[2, num_edges]`
* **Semantic Meaning**: Directed or undirected graph adjacency connections.
* **Equivalent Available**: Yes, `graph.edges`.

### 6. `y`
* **Where it is created**: Formed from target labels (node labels or edge labels).
* **Where it is loaded**: Loaded in `task/node.py` and `task/edge.py`.
* **Where it is consumed**: Used as target class indices for cross-entropy (downstream fine-tuning) or class description lookup (SFT).
* **Mandatory**: Yes.
* **Tensor Dtype**: `torch.long`
* **Tensor Shape**: `[num_nodes]` (for node classification) or `[num_edges]` (for edge classification).
* **Semantic Meaning**: Ground truth labels.
* **Equivalent Available**: Yes, `node.label`.

### 7. `train_mask`, `val_mask`, `test_mask`
* **Where it is created**: Built by partitioning nodes.
* **Where it is loaded**: Loaded in `utils/split.py` and `task/node.py`.
* **Where it is consumed**: Used to select subset of nodes for loss computation and evaluation.
* **Mandatory**: Yes, mandatory for fine-tuning.
* **Tensor Dtype**: `torch.bool`
* **Tensor Shape**: `[num_nodes]`
* **Semantic Meaning**: Boolean flags indicating dataset splits.
* **Equivalent Available**: Yes, generated using random/chronological splits of the graph node indices.

### 8. `groups`
* **Where it is created**: Created for graph-level classification datasets (e.g. molecule graphs).
* **Where it is loaded**: Loaded in `data/finetune_data.py` (multiple_graphs).
* **Where it is consumed**: Consumed in `add_virtual_nodes_graph_classification` to map nodes to graph IDs.
* **Mandatory**: Only mandatory for graph-classification datasets. Not required for our single-graph fraud datasets.
* **Tensor Dtype**: `torch.long`
* **Tensor Shape**: `[num_nodes]`
* **Semantic Meaning**: Maps each node to its respective graph index in a batched representation.
* **Equivalent Available**: Not needed (always None or omitted).

### 9. `prompt_edge_text_feat` and `noi_node_text_feat`
* **Where it is created / loaded / consumed**: Only created and used in `MolOFADataset` for graph-level tasks.
* **Mandatory**: No, not used for single-graph datasets.
* **Tensor Dtype**: `torch.float32`
* **Tensor Shape**: `[num_prompts, embedding_dim]`
* **Semantic Meaning**: Prompts for edge and node classification tasks.
* **Equivalent Available**: Not needed.
