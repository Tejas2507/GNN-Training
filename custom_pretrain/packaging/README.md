# Universal Fraud Graph to GIT Dataset Packager

This package contains the universal converter pipeline that formats standardized `FraudGraph` representations and BGE embeddings into compliance with the ICML 2025 **GIT** (Towards Graph Foundation Models) data loading specifications.

---

## 1. Preprocessing and Packaging Pipeline

The dataset flow operates through the following modular stages:

```text
Raw Dataset (CSV/Pickle)
         ↓  [converters/]
     FraudGraph
         ↓  [text_generation/]
Text Description Generation
         ↓  [multi_gpu_embed.py]
SentenceTransformer Embeddings (npy/json)
         ↓  [packaging/prepare_class_embeddings.py]
Class & Edge Text Embeddings
         ↓  [packaging/universal_converter.py]
Packaged PyTorch Geometric [data]
         ↓
cache_data/<DATASET>/processed/geometric_data_processed.pt
         ↓  [pretrain.py / finetune.py]
GIT Pretraining & Evaluation
```

---

## 2. Directory Layout

The generated artifacts and input paths must follow this structure on the execution environment (e.g. Kaggle):

```text
custom_pretrain/
    cache_output/
        IBM_AML_graph.pkl
        BUPT_graph.pkl
        Elliptic_graph.pkl
    embeddings/
        IBM_AML/
            node_embeddings.npy
            node_ids.json
            class_embeddings.npy       # Generated by prepare_class_embeddings.py
            edge_embeddings.npy        # Generated by prepare_edge_embeddings.py (Optional)
    text_cache/
        IBM_AML/
            class_text.json
            edge_text.json
            node_text.json
    packaging/
        universal_converter.py         # Combines graph, embeddings, classes into a single file
        split_generator.py             # Computes chronological / random split masks
        prepare_class_embeddings.py    # Encodes class category texts to npy array
        prepare_edge_embeddings.py     # Encodes edge texts to npy array (or creates dummy zeros)
        verify_processed.py            # Runs structural assertions on the .pt output
```

---

## 3. Required Inputs and Outputs

### Inputs:
1. **`FraudGraph` pickle file**: `custom_pretrain/cache_output/<DATASET>_graph.pkl`
2. **Node Embeddings**: `custom_pretrain/embeddings/<DATASET>/node_embeddings.npy`
3. **Node IDs List**: `custom_pretrain/embeddings/<DATASET>/node_ids.json`
4. **Class Category Texts**: `custom_pretrain/text_cache/<DATASET>/class_text.json`
5. **Edge Texts (Optional)**: `custom_pretrain/text_cache/<DATASET>/edge_text.json`

### Outputs:
* **Compliant Dataset**: `cache_data/<DATASET>/processed/geometric_data_processed.pt` (a python list containing a single `torch_geometric.data.Data` object).

---

## 4. How to Convert Future Datasets (e.g., PaySim, IEEE-CIS, AMLSim)

To introduce a new dataset to this packaging pipeline without making any changes to the converter code:
1. **Preprocess and Save Graph**: Convert the raw data to a `FraudGraph` and pickle it as `custom_pretrain/cache_output/<NEW_DATASET>_graph.pkl`.
2. **Generate Node Embeddings**: Run your SentenceTransformer script to output `node_embeddings.npy` and `node_ids.json` under `custom_pretrain/embeddings/<NEW_DATASET>/`.
3. **Formulate class_text.json**: Create `custom_pretrain/text_cache/<NEW_DATASET>/class_text.json` mapping target labels to description strings.
4. **Prepare Class/Edge Embeddings**:
   ```bash
   python custom_pretrain/packaging/prepare_class_embeddings.py --dataset <NEW_DATASET>
   python custom_pretrain/packaging/prepare_edge_embeddings.py --dataset <NEW_DATASET> --dummy
   ```
5. **Run the Packaging Step**:
   ```bash
   python custom_pretrain/packaging/universal_converter.py --dataset <NEW_DATASET>
   ```
6. **Verify the Dataset**:
   ```bash
   python custom_pretrain/packaging/verify_processed.py --dataset <NEW_DATASET>
   ```
